# Flask 웹 애플리케이션 - 기업 웹사이트 관리 시스템(암호화된 버전)

## 프로젝트 소개

이는 Flask 프레임워크를 기반으로 개발된 기업 웹사이트 관리 시스템의 **암호화된 버전**입니다. 핵심 코드는 지적 재산권을 보호하기 위해 PyArmor에 의해 난독화되고 암호화되었습니다.이 시스템은 다중 언어(중국어 및 영어), 사용자 인증, 제품 및 뉴스 관리, 회사 정보 표시 및 기타 기능을 지원합니다.

## 출시 노트

이 버전은 완전한 기업 웹사이트 관리 기능을 제공하는 기본 암호화 버전입니다.필요한 경우:

- **완전한 소스 코드 버전**: 2차 개발을 용이하게 하기 위해 암호화되지 않은 완전한 소스 코드가 포함되어 있습니다.
- **맞춤형 기능 개발**: 특정 요구 사항에 따른 기능 맞춤화 및 확장
- **기술 지원 서비스**: 전문적인 기술 지원 및 유지 관리 서비스를 받으세요.

아래 연락처로 연락주시면 자세한 계획과 견적을 안내해드리겠습니다.

## 암호화 기능

- **코드 보호**: 핵심 Python 파일(app.py, init_db.py, models.py)이 난독화되고 암호화됩니다.
- **런타임 보호**: PyArmor 런타임 환경을 사용하여 코드 디컴파일 방지
- **전체 기능**: 암호화된 코드는 원래의 모든 기능과 특징을 유지합니다.
- **다국어 지원**: 중국어 및 영어 전환 기능 내장

## 파일 포함

```
/
├── app.py                  # 암호화된 기본 애플리케이션 항목
├── init_db.py              # 암호화된 데이터베이스 초기화 스크립트
├── models.py               # 암호화된 데이터 모델
├── babel.cfg               # Babel 국제화 구성
├── pyarmor_runtime_000000/ # PyArmor 런타임 지원 파일
├── static/                 # 정적 리소스 파일(CSS, JS, 이미지)
├── templates/              # HTML 템플릿 파일
├── translations/           # 다국어 번역 파일
└── instance/               # 데이터베이스 디렉터리(처음 실행 시 자동으로 생성됨)
```

## 빠른 시작

### 환경 요구사항

- 파이썬 3.9 이상
- 설치된 종속성 패키지:
- 플라스크 3.0.0+
- Flask-SQLAlchemy 3.1.1+
- 플라스크 로그인 0.6.3+
- 플라스크-바벨 4.0.0+
- 작업 3.0.1+

### 종속성 설치

필수 종속성이 아직 설치되지 않은 경우 먼저 설치하십시오.

```bash
pip install flask flask-sqlalchemy flask-login flask-babel werkzeug
```

### 애플리케이션 실행

1. **데이터베이스 초기화**

처음 실행하기 전에 데이터베이스를 초기화해야 합니다.

```bash
   python init_db.py
   ```

2. **애플리케이션 실행**

```bash
   python app.py
   ```

3. **앱 액세스**

브라우저를 열고 http://127.0.0.1:5000을 방문하세요.

## 사용 지침

### 웹사이트 방문

1. 브라우저를 열고 http://127.0.0.1:5000을 방문하세요.
2. 오른쪽 상단의 언어 전환 버튼을 이용하여 중국어와 영어 간 전환이 가능합니다.

### 경영 배경

1. http://127.0.0.1:5000/admin/login을 방문하세요.
2. 관리자 계정으로 로그인합니다. (기본 사용자 이름 및 비밀번호: admin/admin123)
3. 관리백그라운드에서 제품, 뉴스, 회사정보 등을 관리할 수 있습니다.

## 메모

1. **파일 무결성**: 모든 파일, 특히 'pyarmor_runtime_000000' 디렉터리와 해당 콘텐츠가 올바르게 다운로드되었는지 확인하세요.

2. **데이터베이스 파일**:
- 데이터베이스 파일은 'instance' 디렉터리에 자동으로 생성됩니다.
- 데이터 백업이 필요하신 경우 `instance/site.db` 파일을 정기적으로 백업해 주시기 바랍니다.

3. **다국어 지원**:
- 모든 번역 파일은 `translations` 디렉토리에 포함되어 있습니다.
- 새로운 번역 언어를 추가하려면 원본 프로젝트 문서를 참조하세요.

4. **운영 환경**:
- Python 버전이 3.9 이상인지 확인하세요.
- 필요한 모든 종속성이 설치되어 있는지 확인하십시오.

5. **안전 팁**:
- 프로덕션 환경에 배포 시 기본 관리자 비밀번호를 변경하시기 바랍니다.
- 개발 서버 대신 WSGI 서버(예: Gunicorn) 사용을 고려하세요.
- 적절한 방화벽 규칙 구성

## 배포 권장사항

### 개발 환경

Flask 내장 개발 서버를 사용합니다(위의 빠른 시작에 표시됨).

### 제작 환경

1. **WSGI 서버 사용**:
```bash
   pip install gunicorn
   gunicorn -w 4 -b 0.0.0.0:8000 app:app
   ```

2. **역방향 프록시 사용**:
- Nginx 또는 Apache를 역방향 프록시로 구성
- HTTPS를 활성화하려면 SSL 인증서를 설정하세요.

3. **데이터베이스 최적화**:
- SQLite 대신 PostgreSQL 또는 MySQL 사용을 고려하세요.
- 정기적인 백업 메커니즘 구성

## 문제 해결

### FAQ

1. **애플리케이션을 시작할 수 없습니다**:
- Python 버전이 요구사항을 충족하는지 확인하세요.
- 모든 종속 패키지가 올바르게 설치되었는지 확인
- `pyarmor_runtime_000000` 디렉터리가 존재하고 완전한지 확인하세요.

2. **언어 전환이 작동하지 않음**:
- `translations` 디렉토리와 그 내용이 완전한지 확인하세요.
- 브라우저 쿠키 설정이 허용되는지 확인하세요.

3. **데이터베이스 연결 오류**:
- 'instance' 디렉터리가 존재하고 쓰기 권한이 있는지 확인하세요.
- 'init_db.py'를 다시 실행하여 데이터베이스를 초기화해 보세요.
