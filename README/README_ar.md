# تطبيق Flask Web - نظام إدارة موقع المؤسسة (نسخة مشفرة)

## مقدمة المشروع

هذه **نسخة مشفرة** من نظام إدارة موقع الويب الخاص بالمؤسسة والذي تم تطويره استنادًا إلى إطار عمل Flask. تم حجب الكود الأساسي وتشفيره بواسطة PyArmor لحماية حقوق الملكية الفكرية.يدعم النظام متعدد اللغات (الصينية والإنجليزية)، ومصادقة المستخدم، وإدارة المنتجات والأخبار، وعرض معلومات الشركة ووظائف أخرى.

## ملاحظات الإصدار

هذا الإصدار هو إصدار تشفير أساسي يوفر وظائف كاملة لإدارة موقع الشركة على الويب.إذا كنت بحاجة إلى:

- **إصدار كود المصدر الكامل**: يحتوي على كود مصدر كامل غير مشفر لتسهيل التطوير الثانوي
- **تطوير الوظائف المخصصة**: تخصيص الوظائف وتوسيعها وفقًا لاحتياجاتك المحددة
- **خدمات الدعم الفني**: احصل على خدمات الدعم الفني والصيانة الاحترافية

يرجى الاتصال بنا من خلال معلومات الاتصال أدناه وسنزودك بالخطط التفصيلية وعروض الأسعار.

## مميزات التشفير

- **حماية التعليمات البرمجية**: ملفات Python الأساسية (app.py وinit_db.py وmodels.py) مشوشة ومشفرة
- **حماية وقت التشغيل**: استخدم بيئة تشغيل PyArmor لمنع فك التعليمات البرمجية
- **الوظائف الكاملة**: يحتفظ الرمز المشفر بجميع الوظائف والميزات الأصلية
- **دعم متعدد اللغات**: وظيفة التبديل الصينية والإنجليزية المدمجة

## تضمين الملفات

```
/
├── app.py                  # إدخال التطبيق الرئيسي المشفر
├── init_db.py              # البرنامج النصي لتهيئة قاعدة البيانات المشفرة
├── models.py               # نماذج البيانات المشفرة
├── babel.cfg               # تكوين تدويل بابل
├── pyarmor_runtime_000000/ # ملفات دعم وقت التشغيل PyArmor
├── static/                 # ملفات الموارد الثابتة (CSS، JS، الصور)
├── templates/              # ملفات قالب HTML
├── translations/           # ملفات ترجمة متعددة اللغات
└── instance/               # دليل قاعدة البيانات (يتم إنشاؤه تلقائيًا عند التشغيل لأول مرة)
```

## بداية سريعة

### المتطلبات البيئية

- بايثون 3.9 أو أعلى
- حزم التبعية المثبتة:
- قارورة 3.0.0+
- قارورة SQLAlchemy 3.1.1+
- قارورة تسجيل الدخول 0.6.3+
- قارورة بابل 4.0.0+
- العمل 3.0.1+

### تثبيت التبعيات

إذا لم يتم تثبيت التبعيات المطلوبة بعد، فيرجى تثبيتها أولاً:

```bash
pip install flask flask-sqlalchemy flask-login flask-babel werkzeug
```

### قم بتشغيل التطبيق

1. **تهيئة قاعدة البيانات**

قبل التشغيل لأول مرة، يجب تهيئة قاعدة البيانات:

```bash
   python init_db.py
   ```

2. ** إطلاق التطبيق **

```bash
   python app.py
   ```

3. **تطبيق الوصول**

افتح المتصفح وقم بزيارة: http://127.0.0.1:5000

## تعليمات الاستخدام

### قم بزيارة الموقع

1. افتح المتصفح وقم بزيارة http://127.0.0.1:5000
2. استخدم زر تبديل اللغة في الزاوية اليمنى العليا للتبديل بين الصينية والإنجليزية

### خلفية الإدارة

1. قم بزيارة http://127.0.0.1:5000/admin/login
2. قم بتسجيل الدخول باستخدام حساب المسؤول (اسم المستخدم وكلمة المرور الافتراضية: admin/admin123)
3. يمكن إدارة المنتجات والأخبار ومعلومات الشركة في خلفية الإدارة

## وصف الوظيفة

### وظائف الواجهة الأمامية

- **الصفحة الرئيسية**: عرض ملف الشركة والمنتجات الرئيسية
- **نبذة عنا**: عرض تفاصيل الشركة
- **عرض المنتج**: تصفح جميع قوائم المنتجات
- **تحديثات الصناعة**: عرض آخر الأخبار والمعلومات
- **تبديل اللغة**: يدعم التبديل بين الصينية والإنجليزية

### إدارة وظائف الخلفية

- **إدارة المنتجات**: إضافة المنتجات وتعديلها وحذفها
- **إدارة الأخبار**: نشر الأخبار وتعديلها وحذفها
- **معلومات الشركة**: تحديث معلومات الشركة الأساسية
- **مصادقة المستخدم**: نظام تسجيل دخول آمن

## دليل لقطة الشاشة

للحصول على أفضل تأثير لعرض README، يوصى بإنشاء لقطات شاشة وإضافتها وفقًا للإرشادات التالية:

1. **إنشاء مجلد لقطات الشاشة**: قم بإنشاء المجلد `لقطات الشاشة/` في الدليل `dist/`
2. **حجم لقطة الشاشة**: استخدم لقطة شاشة بدقة 1920×1080 أو 1366×768 لضمان رؤية المحتوى بوضوح
3. **محتوى لقطة الشاشة**:
- الصفحة الرئيسية: تعرض واجهة الصفحة الرئيسية الكاملة، بما في ذلك شريط التنقل ومنطقة المحتوى الرئيسي
- تبديل اللغة: إظهار التأثير المقارن للتبديل بين الصينية والإنجليزية (يمكن استخدام لقطات شاشة مقسمة)
- عرض المنتج: عرض قائمة المنتجات وتفاصيل المنتجات الفردية
- صفحة الأخبار: عرض قائمة الأخبار وتفاصيل الأخبار
- الواجهة الخلفية للإدارة: تعرض لوحة المعلومات والواجهة الوظيفية بعد تسجيل دخول المسؤول
4. **تنسيق لقطة الشاشة**: استخدم تنسيق PNG للحصول على أفضل جودة
5. **تسمية الملف**: احفظ لقطة الشاشة وفقًا لاسم الملف المحدد في الملف التمهيدي (index.png، language_switch.png، وما إلى ذلك)

## ملاحظات

1. **سلامة الملف**: يرجى التأكد من تنزيل جميع الملفات بشكل صحيح، وخاصة الدليل `pyarmor_runtime_000000` ومحتوياته

2. **ملف قاعدة البيانات**:
- سيتم إنشاء ملف قاعدة البيانات تلقائيًا في دليل "المثيل".
- إذا كنت بحاجة إلى نسخ البيانات احتياطيًا، فيرجى عمل نسخة احتياطية من ملف "instance/site.db" بانتظام

3. ** دعم متعدد اللغات **:
- جميع ملفات الترجمة موجودة في دليل "الترجمات".
- لإضافة لغات ترجمة جديدة، يرجى الرجوع إلى وثائق المشروع الأصلية

4. ** بيئة التشغيل **:
- تأكد من أن إصدار بايثون لا يقل عن 3.9
- تأكد من تثبيت كافة التبعيات الضرورية

5. **نصائح للسلامة**:
- عند النشر في بيئة إنتاج، يرجى تغيير كلمة مرور المسؤول الافتراضية.
- فكر في استخدام خادم WSGI (مثل Gunicorn) بدلاً من خادم التطوير
- تكوين قواعد جدار الحماية المناسبة

## توصيات النشر

### بيئة التطوير

استخدم خادم التطوير المدمج Flask (كما هو موضح في البداية السريعة أعلاه).

### بيئة الإنتاج

1. **استخدم خادم WSGI**:
```bash
   pip install gunicorn
   gunicorn -w 4 -b 0.0.0.0:8000 app:app
   ```

2. **استخدم الوكيل العكسي**:
- قم بتكوين Nginx أو Apache كوكيل عكسي
- قم بإعداد شهادة SSL لتمكين HTTPS

3. **تحسين قاعدة البيانات**:
- فكر في استخدام PostgreSQL أو MySQL بدلاً من SQLite
- تكوين آلية النسخ الاحتياطي العادية

## استكشاف الأخطاء وإصلاحها

### الأسئلة الشائعة

1. **لا يمكن بدء التطبيق**:
- التحقق مما إذا كان إصدار Python يلبي المتطلبات
- تأكد من تثبيت كافة الحزم التابعة بشكل صحيح
- تحقق مما إذا كان الدليل `pyarmor_runtime_000000` موجودًا ومكتملًا

2. **تبديل اللغة لا يعمل**:
- التأكد من اكتمال دليل "الترجمات" ومحتوياته
- التحقق مما إذا كانت إعدادات ملفات تعريف الارتباط للمتصفح مسموحة أم لا

3. **خطأ في الاتصال بقاعدة البيانات**:
- تأكد من وجود دليل "المثيل" ولديه أذونات الكتابة
- حاول إعادة تشغيل "init_db.py" لتهيئة قاعدة البيانات

### عرض السجل

عندما يبدأ التطبيق، سيتم إخراج معلومات السجل على وحدة التحكم. إذا واجهت مشاكل، يمكنك التحقق من هذه السجلات للحصول على مزيد من المعلومات.
