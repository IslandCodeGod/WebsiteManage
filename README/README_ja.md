# Flask Web Application - エンタープライズ Web サイト管理システム (暗号化バージョン)

## プロジェクトの紹介

これは、Flask フレームワークに基づいて開発されたエンタープライズ Web サイト管理システムの **暗号化バージョン** です。コア コードは、知的財産権を保護するために PyArmor によって難読化および暗号化されています。多言語（中国語、英語）対応、ユーザー認証、商品・ニュース管理、企業情報表示などの機能をサポートしています。

## リリースノート

このバージョンは、完全な企業 Web サイト管理機能を提供する基本的な暗号化バージョンです。必要な場合:

- **完全なソース コード バージョン**: 二次開発を容易にする、暗号化されていない完全なソース コードが含まれています。
- **カスタマイズされた機能開発**: 特定のニーズに応じた機能のカスタマイズと拡張
- **テクニカル サポート サービス**: 専門的なテクニカル サポートとメンテナンス サービスを利用できます。

下記お問い合わせ先までお問い合わせいただければ、詳細なプランやお見積りをご案内させていただきます。

## 暗号化機能

- **コード保護**: コア Python ファイル (app.py、init_db.py、models.py) は難読化および暗号化されています。
- **ランタイム保護**: PyArmor ランタイム環境を使用して、コードが逆コンパイルされるのを防ぎます。
- **完全な機能**: 暗号化されたコードは、元の機能をすべて維持します。
- **多言語サポート**: 中国語と英語の切り替え機能を内蔵

## インクルードファイル

```
/
├── app.py                  # 暗号化されたメインアプリケーションエントリ
├── init_db.py              # 暗号化されたデータベース初期化スクリプト
├── models.py               # 暗号化されたデータモデル
├── babel.cfg               # Babel 国際化構成
├── pyarmor_runtime_000000/ # PyArmor ランタイム サポート ファイル
├── static/                 # 静的リソース ファイル (CSS、JS、画像)
├── templates/              # HTMLテンプレートファイル
├── translations/           # 多言語翻訳ファイル
└── instance/               # データベース ディレクトリ (初回実行時に自動的に作成されます)
```

## クイックスタート

### 環境要件

- Python 3.9以降
- インストールされている依存関係パッケージ:
- フラスコ 3.0.0+
- Flask-SQLAlchemy 3.1.1+
- Flask-ログイン 0.6.3+
- Flask-Babel 4.0.0+
- ヴェルクツァーク 3.0.1+

### 依存関係をインストールする

必要な依存関係がまだインストールされていない場合は、最初にインストールしてください。

```bash
pip install flask flask-sqlalchemy flask-login flask-babel werkzeug
```

### アプリケーションを実行する

1. **データベースの初期化**

初めて実行する前に、データベースを初期化する必要があります。

```bash
   python init_db.py
   ```

2. **アプリケーションを起動します**

```bash
   python app.py
   ```

3. **アプリにアクセス**

ブラウザを開いて、http://127.0.0.1:5000 にアクセスします。

## 注意事項

1. **ファイルの整合性**: すべてのファイル、特に「pyarmor_runtime_000000」ディレクトリとその内容が正しくダウンロードされていることを確認してください。

2. **データベース ファイル**:
- データベース ファイルは `instance` ディレクトリに自動的に作成されます
- データをバックアップする必要がある場合は、「instance/site.db」ファイルを定期的にバックアップしてください。

3. **多言語サポート**:
- すべての翻訳ファイルは「translations」ディレクトリに含まれます
- 新しい翻訳言語を追加するには、元のプロジェクトのドキュメントを参照してください。

4. **動作環境**:
- Python のバージョンが 3.9 以上であることを確認してください。
- 必要な依存関係がすべてインストールされていることを確認してください

5. **安全に関するヒント**:
・ 本番環境に導入する場合は、デフォルトの管理者パスワードを変更してください。
- 開発サーバーの代わりに WSGI サーバー (Gunicorn など) の使用を検討してください。
- 適切なファイアウォール ルールを構成する

1. **WSGI サーバーを使用します**:

- Nginx または Apache をリバース プロキシとして構成する

- SQLite の代わりに PostgreSQL または MySQL の使用を検討してください。
- 定期的なバックアップメカニズムを構成する

## トラブルシューティング

### よくある質問

1. **アプリケーションを開始できません**:
- Pythonのバージョンが要件を満たしているか確認します
- すべての依存パッケージが正しくインストールされていることを確認します
- `pyarmor_runtime_000000` ディレクトリが存在し、完了しているかどうかを確認します

2. **言語切り替えが機能しない**:
- 「translations」ディレクトリとその内容が完了していることを確認します。
- ブラウザのCookie設定が許可されているかどうかを確認してください

3. **データベース接続エラー**:
- 「instance」ディレクトリが存在し、書き込み権限があることを確認します。
- `init_db.py` を再実行してデータベースを初期化してください。

### ログビュー

アプリケーションが起動すると、コンソールにログ情報が出力されます。問題が発生した場合は、これらのログで詳細を確認できます。
