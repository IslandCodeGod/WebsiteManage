# Веб-приложение Flask — система управления корпоративным веб-сайтом (зашифрованная версия)

## Введение в проект

Это **зашифрованная версия** системы управления корпоративными веб-сайтами, разработанная на основе платформы Flask. Основной код был запутан и зашифрован PyArmor для защиты прав интеллектуальной собственности.Система поддерживает многоязычность (китайский и английский), аутентификацию пользователей, управление продуктами и новостями, отображение информации о компании и другие функции.

## Функции шифрования

- **Защита кода**: основные файлы Python (app.py, init_db.py, models.py) скрыты и зашифрованы.
- **Защита во время выполнения**: используйте среду выполнения PyArmor, чтобы предотвратить декомпиляцию кода.
- **Полная функциональность**: зашифрованный код сохраняет все исходные функции и возможности.
- **Многоязычная поддержка**: встроенная функция переключения китайского и английского языков.

## Включить файлы

```
/
├── app.py                  # Зашифрованная запись основного приложения
├── init_db.py              # Скрипт инициализации зашифрованной базы данных
├── models.py               # Модели зашифрованных данных
├── babel.cfg               # Конфигурация интернационализации Babel
├── pyarmor_runtime_000000/ # Файлы поддержки среды выполнения PyArmor
├── static/                 # Статические файлы ресурсов (CSS, JS, изображения)
├── templates/              # HTML-файлы шаблонов
├── translations/           # Файлы многоязычного перевода
└── instance/               # Каталог базы данных (автоматически создается при первом запуске)
```

## Быстрый старт

### Экологические требования

- Питон 3.9 или выше
- Установленные пакеты зависимостей:
- Колба 3.0.0+
- Flask-SQLAlchemy 3.1.1+
- Flask-Вход 0.6.3+
- Фляга-Вавилон 4.0.0+
- Веркцойг 3.0.1+

### Установка зависимостей

Если необходимые зависимости еще не установлены, сначала установите их:

```bash
pip install flask flask-sqlalchemy flask-login flask-babel werkzeug
```

### Запустите приложение

1. **Инициализировать базу данных**

Перед первым запуском базу данных необходимо инициализировать:

```bash
   python init_db.py
   ```

2. **Запустите приложение**

```bash
   python app.py
   ```

3. **Доступ к приложению**

Откройте браузер и посетите: http://127.0.0.1:5000.

## Инструкция по применению

### Посетите веб-сайт

1. Откройте браузер и посетите http://127.0.0.1:5000.
2. Используйте кнопку переключения языка в правом верхнем углу, чтобы переключиться между китайским и английским языками.

### Опыт управления

1. Посетите http://127.0.0.1:5000/admin/login.
2. Войдите в систему под учетной записью администратора (имя пользователя и пароль по умолчанию: admin/admin123).
3. Продуктами, новостями и информацией о компании можно управлять в фоновом режиме управления.

## Примечания

1. **Целостность файла**: убедитесь, что все файлы загружены правильно, особенно каталог `pyarmor_runtime_000000` и его содержимое.

2. **Файл базы данных**:
- Файл базы данных будет автоматически создан в каталоге экземпляра.
- Если вам необходимо выполнить резервное копирование данных, регулярно создавайте резервную копию файла `instance/site.db`.

3. **Многоязычная поддержка**:
- Все файлы переводов включены в каталог `translations`.
- Чтобы добавить новые языки перевода, обратитесь к исходной документации проекта.

4. **Рабочая среда**:
- Убедитесь, что версия Python не ниже 3.9.
- Убедитесь, что установлены все необходимые зависимости.

5. **Советы по безопасности**:
- При развертывании в производственной среде измените пароль администратора по умолчанию.
– Рассмотрите возможность использования сервера WSGI (например, Gunicorn) вместо сервера разработки.
- Настройте соответствующие правила брандмауэра.

## Рекомендации по развертыванию

### Среда разработки

Используйте встроенный сервер разработки Flask (показан в кратком руководстве выше).

### Производственная среда

1. **Использовать сервер WSGI**:
```bash
   pip install gunicorn
   gunicorn -w 4 -b 0.0.0.0:8000 app:app
   ```

2. **Использовать обратный прокси**:
- Настройте Nginx или Apache в качестве обратного прокси-сервера.
- Настройка SSL-сертификата для включения HTTPS.

3. **Оптимизация базы данных**:
- Рассмотрите возможность использования PostgreSQL или MySQL вместо SQLite.
- Настройка регулярного механизма резервного копирования.

## Устранение неполадок

### Часто задаваемые вопросы

1. **Приложение не может быть запущено**:
- Проверьте, соответствует ли версия Python требованиям.
- Убедитесь, что все зависимые пакеты установлены правильно.
- Проверьте, существует ли каталог `pyarmor_runtime_000000` и является ли он полным.

2. **Переключение языка не работает**:
- Убедитесь, что каталог `translations` и его содержимое заполнены.
- Проверьте, разрешены ли настройки файлов cookie браузера.

3. **Ошибка подключения к базе данных**:
- Убедитесь, что каталог `экземпляра` существует и имеет разрешения на запись.
- Попробуйте повторно запустить `init_db.py`, чтобы инициализировать базу данных.
