# Flask Web应用 - 企业网站管理系统（加密版本）

## 项目简介

这是一个基于Flask框架开发的企业网站管理系统的**加密版本**，核心代码经过PyArmor混淆加密处理，保护知识产权。系统支持多语言（中英文）、用户认证、产品与新闻管理、公司信息展示等功能。

## 快速开始

### 环境要求

- Python 3.9 或更高版本
- 已安装的依赖包：
  - Flask 3.0.0+
  - Flask-SQLAlchemy 3.1.1+
  - Flask-Login 0.6.3+
  - Flask-Babel 4.0.0+
  - Werkzeug 3.0.1+

### 安装依赖

如果尚未安装所需依赖，请先安装：

```bash
pip install flask flask-sqlalchemy flask-login flask-babel werkzeug
```

### 运行应用

1. **初始化数据库**

   首次运行前，需要初始化数据库：

   ```bash
   python init_db.py
   ```

2. **启动应用**

   ```bash
   python app.py
   ```

3. **访问应用**

   打开浏览器，访问：http://127.0.0.1:5000

## 注意事项

1. **文件完整性**：请确保所有文件都已正确下载，特别是`pyarmor_runtime_000000`目录及其内容

2. **数据库文件**：
   - 数据库文件会自动在`instance`目录下创建
   - 如需备份数据，请定期备份`instance/site.db`文件

3. **多语言支持**：
   - 所有翻译文件已包含在`translations`目录中
   - 如需添加新的翻译语言，请参考原始项目文档

4. **运行环境**：
   - 确保Python版本不低于3.9
   - 确保已安装所有必要的依赖包

5. **安全提示**：
   - 生产环境部署时，请修改默认管理员密码
   - 考虑使用WSGI服务器（如Gunicorn）代替开发服务器
   - 配置适当的防火墙规则

## 部署建议

### 开发环境

使用Flask内置开发服务器（如上述快速开始所示）。

### 生产环境

1. **使用WSGI服务器**：
   ```bash
   pip install gunicorn
   gunicorn -w 4 -b 0.0.0.0:8000 app:app
   ```

2. **使用反向代理**：
   - 配置Nginx或Apache作为反向代理
   - 设置SSL证书启用HTTPS

3. **数据库优化**：
   - 考虑使用PostgreSQL或MySQL替代SQLite
   - 配置定期备份机制

## 故障排除

### 常见问题

1. **应用无法启动**：
   - 检查Python版本是否符合要求
   - 确认所有依赖包已正确安装
   - 检查`pyarmor_runtime_000000`目录是否存在且完整

2. **语言切换不工作**：
   - 确认`translations`目录及其内容完整
   - 检查浏览器Cookies设置是否允许

3. **数据库连接错误**：
   - 确认`instance`目录存在且具有写入权限
   - 尝试重新运行`init_db.py`初始化数据库
